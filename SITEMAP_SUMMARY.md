# Sitemap Implementation Summary

Last updated: 2026-05-14

## Implementation Status

The site uses a generated static sitemap index and two generated sitemap files.

Current production sitemap paths:

- `/sitemap.xml`
- `/sitemaps/0.xml`
- `/sitemaps/1.xml`

The older `/sitemap/0.xml` and `/sitemap/1.xml` paths are legacy paths and should not be used for current verification. The proxy redirects older sitemap patterns back to `/sitemap.xml`.

## How Sitemaps Are Generated

Sitemaps are generated by:

```bash
npm run generate:sitemaps
```

The production prebuild hook runs:

```bash
npm run generate:redirects && npm run generate:sitemaps:if-production
```

The generator writes:

- `public/sitemap.xml`
- `public/sitemaps/0.xml`
- `public/sitemaps/1.xml`

Relevant files:

- `scripts/generate-sitemaps.ts`
- `scripts/run-generate-sitemaps.cjs`
- `src/lib/sitemap-helpers.ts`
- `src/lib/sitemap-config.ts`
- `src/lib/sitemap-utils.ts`
- `src/app/robots.ts`

## Why Counts Are Lower Than Older Docs

Older documentation listed much higher sitemap totals. Those numbers came from an earlier strategy that submitted many more location-service combinations.

The current code intentionally filters location-service pages through `isIndexableLocationService` in `src/lib/seo-data.ts`. This keeps the sitemap focused on stronger commercial targets and avoids submitting large groups of duplicate or thin location-service pages.

## Verification Commands

Compile-check the sitemap generator:

```bash
npm run generate:sitemaps -- --check
```

Print current sitemap counts:

```bash
npx tsx -e "import { printSitemapCounts } from './src/lib/sitemap-utils'; printSitemapCounts();"
```

Audit SEO architecture:

```bash
npm run audit:seo-architecture
```

When a local server is running, verify live redirects and page behavior:

```bash
npm run audit:seo-architecture -- --base-url http://localhost:3000
```
